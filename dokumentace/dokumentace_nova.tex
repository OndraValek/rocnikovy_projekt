%% Abstrakt
	\noindent{\Large{\bfseries{Abstrakt}\\}}
	\noindent Výsledkem projektu je funkční webová aplikace pro~přípravu studentů k~maturitním zkouškám z~předmětu Programové vybavení. Aplikace zahrnuje registraci uživatelů přes OAuth2 (Google, GitHub, Microsoft), včetně standardních operací s~uživatelským účtem. Uživatelé si mohou prohlížet studijní materiály, řešit interaktivní testy vytvořené v~H5P a~účastnit se diskusního fóra. Stěžejní částí aplikace je systém statistik, který studentům zobrazuje přehled jejich výsledků v~testech včetně grafického zobrazení a~automatického známkování. Učitelé a~administrátoři mohou spravovat předměty, okruhy, materiály a~testy. Aplikace má zcela responzivní design a~funguje na~všech zařízeních. Aplikace je postavena na~Django frameworku s~Wagtail CMS, využívá PostgreSQL databázi a~je připravena pro~nasazení pomocí Dockeru.
	
	\vspace{18pt}
	
	\noindent{\large{\bfseries{Klíčová slova}}}
	
	\noindent webová aplikace, Django, Wagtail CMS, H5P, OAuth2, PostgreSQL, responzivní design, vzdělávací platforma
	
	\vspace{18pt}

	\noindent{\Large{\bfseries{Abstract}}}
	
	\noindent The result of this project is a functional web application for preparing students for final exams in the subject of Programming Equipment. The application includes user registration via OAuth2 (Google, GitHub, Microsoft), including standard user account operations. Users can browse study materials, solve interactive tests created in H5P, and participate in a discussion forum. The core part of the application is a statistics system that displays an overview of students' test results including graphical representation and automatic grading. Teachers and administrators can manage subjects, topics, materials, and tests. The application has a fully responsive design and works on all devices. The application is built on the Django framework with Wagtail CMS, uses PostgreSQL database, and is ready for deployment using Docker.
	
	\vspace{18pt}
	
	\noindent{\large{\bfseries{Keywords}}}
	
	\noindent web application, Django, Wagtail CMS, H5P, OAuth2, PostgreSQL, responsive design, educational platform

	\chapter*{Úvod}
	\addcontentsline{toc}{chapter}{Úvod}
	
	Příprava k~maturitní zkoušce vyžaduje systematický přístup a~přehledný systém pro~organizaci studijních materiálů, testů a~sledování pokroku. Tradiční metody přípravy pomocí papírových poznámek a~tištěných materiálů jsou v~dnešní době nahrazovány moderními elektronickými řešeními, která nabízejí větší flexibilitu, interaktivitu a~možnost okamžité zpětné vazby.
	
	Mým cílem bylo vytvoření komplexní webové aplikace, která by studentům poskytla přehledný systém pro~přípravu k~maturitě z~předmětu Programové vybavení. Aplikace by měla obsahovat registraci a~přihlášení uživatelů včetně OAuth2 autentizace, umožnit správu studijních materiálů a~interaktivních testů vytvořených v~H5P, poskytovat statistické přehledy výsledků s~grafickým zobrazením a~automatickým známkováním, a~nabízet diskusní fórum pro~komunikaci mezi studenty a~učiteli.
	
	Celá webová aplikace by měla být plně responzivní, aby fungovala na~různých zařízeních od~desktopových počítačů až po~mobilní telefony. Aplikace by také měla být připravena pro~snadné nasazení pomocí Dockeru.
	
	V~této dokumentaci podrobně popisuji postup vytvoření dané aplikace. V~úvodu se zabývám návrhem databázového modelu a~architekturou aplikace. Následně řeším uživatelský systém s~OAuth2 autentizací, integraci H5P obsahu, systém statistik a~správu obsahu. V~závěru popisuji výsledky projektu a~možnosti dalšího rozvoje.

	\chapter{Teoretická a metodická východiska}

	\section{Webové vzdělávací platformy a architektura aplikace}
	
	Moderní vzdělávací platformy (Learning Management Systems~--~LMS) poskytují komplexní řešení pro~správu výukového obsahu, hodnocení studentů a~komunikaci. V~rámci tohoto projektu jsem se zaměřil na~vytvoření specializované platformy pro~přípravu k~maturitním zkouškám, která kombinuje prvky LMS s~moderními webovými technologiemi a~interaktivním obsahem.
	
	Pro~realizaci této platformy jsem zvolil architekturu Model-View-Template (MVT), kterou implementuje Django framework. Tato architektura umožňuje oddělení logiky aplikace od~prezentace, což zajišťuje lepší udržovatelnost a~škálovatelnost kódu. Django poskytuje množství vestavěných funkcí pro~autentizaci, správu databáze, administrační rozhraní a~další, což výrazně urychluje vývoj. Pro~ukládání dat jsem zvolil PostgreSQL, který je ideální volbou pro~produkční prostředí díky své spolehlivosti, výkonu a~podpoře pokročilých funkcí, jako je ACID compliance, pokročilé datové typy a~full-textové vyhledávání. Pro~vývoj jsem použil SQLite, což zjednodušuje nastavení vývojového prostředí.
	
	Pro~správu obsahu aplikace jsem integroval Wagtail CMS, který poskytuje flexibilní systém pro~správu strukturovaného obsahu. Wagtail umožňuje vytvářet stránky s~různými typy bloků obsahu pomocí StreamField, což je ideální pro~vzdělávací obsah. Hlavní výhoda Wagtailu spočívá v~jeho flexibilním systému StreamField, který umožňuje vytvářet stránky s~různými typy bloků obsahu (text, obrázky, H5P obsah, videa atd.) bez~nutnosti programování. Wagtail také poskytuje integraci vlastních Django modelů do~redakčního rozhraní pomocí ModelViewSet API, což umožňuje správu těchto entit přímo z~Wagtail adminu. Wagtail tedy slouží jako vrstva mezi databázovými modely a~uživatelským rozhraním, která umožňuje neprogramátorům (učitelům) spravovat obsah aplikace pomocí intuitivního redakčního rozhraní.

	\section{Interaktivní vzdělávací obsah a uživatelské rozhraní}
	
	H5P (HTML5 Package) je open-source framework pro~vytváření interaktivního vzdělávacího obsahu. Umožňuje vytvářet interaktivní videa, kvízy, prezentace a~další typy obsahu, které lze snadno integrovat do~webových aplikací. Pro~tento projekt jsem zvolil H5P Standalone Player, který umožňuje zobrazovat H5P obsah bez~nutnosti externího H5P serveru. Toto řešení poskytuje plnou kontrolu nad~zobrazením a~ukládáním výsledků, což je klíčové pro~systém statistik. H5P Standalone Player je distribuován jako sada JavaScript a~CSS souborů, které lze zahrnout přímo do~projektu. H5P soubory (.h5p) jsou ve~skutečnosti ZIP archivy obsahující JSON konfigurační soubory, JavaScript, CSS a~další zdroje. Standalone Player tyto soubory rozbalí a~zobrazí pomocí vlastního renderovacího engine.
	
	Pro~zajištění responzivního designu a~moderního uživatelského rozhraní jsem použil Bootstrap~5, front-end framework pro~rychlý vývoj responzivních webových aplikací. Bootstrap poskytuje předpřipravené CSS třídy a~JavaScript komponenty pro~vytváření moderních uživatelských rozhraní. Grid systém umožňuje vytvářet responzivní layouty, které se automaticky přizpůsobují různým velikostem obrazovek, což zajišťuje, že~aplikace vypadá dobře a~funguje na~všech zařízeních od~desktopových počítačů až po~mobilní telefony.
	
	Pro~zobrazení statistik výsledků testů studentů jsem použil Chart.js, JavaScriptovou knihovnu pro~vytváření grafů a~diagramů. Chart.js poskytuje jednoduché API pro~vytváření různých typů grafů (sloupcové, čárové, koláčové atd.) s~možností interakce. Je to lehká knihovna, která nevyžaduje závislosti jako jQuery. V~projektu jsem použil sloupcové grafy pro~zobrazení výsledků testů, které jsou dynamicky generovány pomocí JavaScriptu na~základě dat získaných z~API.

	\chapter{Způsoby řešení a použité postupy}

	\section{Struktura projektu}
	
	Projekt byl vytvořen pomocí standardního Django projektu s~modulární strukturou, která umožňuje snadnou údržbu a~rozšiřitelnost. Aplikace je rozdělena do~několika samostatných modulů:
	
	\begin{itemize}
		\item \textbf{accounts}~--~správa uživatelských účtů a~rolí (Student, Učitel, Admin).
		\item \textbf{subjects}~--~správa předmětů a~maturitních okruhů.
		\item \textbf{materials}~--~správa výukových materiálů různých typů.
		\item \textbf{quizzes}~--~správa testů a~kvízů s~H5P integrací.
		\item \textbf{forum}~--~diskusní fórum pro~každý okruh.
		\item \textbf{maturitni\_projekt}~--~hlavní konfigurace projektu včetně Wagtail CMS.
	\end{itemize}
	
	Každá aplikace obsahuje modely (models.py), views (views.py), URL konfiguraci (urls.py) a~šablony v~adresáři templates. Statické soubory (CSS, JavaScript) jsou umístěny v~adresáři static, zatímco uživatelsky nahrané soubory (H5P soubory, PDF dokumenty, obrázky) jsou ukládány v~adresáři media.

	\section{Databázový model}
	
	Databázový model aplikace je navržen tak, aby podporoval hierarchickou strukturu předmětů a~okruhů, různé typy materiálů a~testů, a~sledování pokusů studentů. Model je implementován pomocí Django ORM, který zajišťuje abstrakci nad~databázovými dotazy a~zjednodušuje práci s~daty.
	
	Hlavní modely aplikace:
	\begin{itemize}
		\item \textbf{User}~--~rozšířený uživatelský model s~rolemi (Student, Učitel, Admin), který rozšiřuje Django User model o~dodatečné informace.
		\item \textbf{Subject}~--~maturitní předmět obsahující název, slug, popis a~seznam okruhů.
		\item \textbf{Topic}~--~okruh v~rámci předmětu, který obsahuje materiály, testy a~diskusní fórum.
		\item \textbf{Material}~--~výukový materiál různých typů (PDF, video, H5P, odkaz, text, obrázek).
		\item \textbf{Quiz}~--~test/kvíz s~H5P integrací, obsahující parametry jako časový limit, maximální počet pokusů a~passing score.
		\item \textbf{QuizAttempt}~--~pokus studenta o~vyplnění testu, obsahující skóre, datum a~čas dokončení.
		\item \textbf{ForumThread}~--~diskusní vlákno v~rámci okruhu.
		\item \textbf{ForumPost}~--~příspěvek ve~fóru, který může být upravován a~mazán autorem.
	\end{itemize}
	
	Vztahy mezi modely jsou navrženy tak, aby podporovaly hierarchickou strukturu a~umožňovaly efektivní dotazování:
	\begin{itemize}
		\item Subject má více Topic (one-to-many vztah).
		\item Topic má více Material a~Quiz (one-to-many vztah).
		\item Quiz má více QuizAttempt (one-to-many vztah).
		\item User má více QuizAttempt (one-to-many vztah).
		\item ForumThread má více ForumPost (one-to-many vztah).
	\end{itemize}
	
	Tato struktura umožňuje efektivní správu obsahu a~sledování pokroku studentů v~rámci jednotlivých předmětů a~okruhů.

	\section{Backend aplikace}
	
	Backend aplikace je postaven na~Django frameworku a~poskytuje REST API pro~frontend, zpracovává obchodní logiku a~spravuje databázové operace. Hlavní komponenty backendu zahrnují modely, views, URL routing a~middleware.
	
	\subsection{Autentizace a autorizace}
	
	Pro~autentizaci jsem použil balíček django-allauth, který poskytuje komplexní řešení pro~autentizaci, registraci a~správu účtů. Django-allauth poskytuje hotové šablony a~views pro~registraci, přihlášení, odhlášení, změnu hesla, reset hesla a~ověření emailu. Pro~OAuth2 autentizaci stačí nakonfigurovat Social Applications v~Django adminu s~client ID a~secret key od~poskytovatele. Django-allauth pak automaticky zpracuje celý OAuth2 flow včetně redirectů a~získání uživatelských dat.
	
	V~projektu jsem využil django-allauth pro~implementaci autentizace přes Google, GitHub a~Microsoft. OAuth2 credentials jsou uloženy v~.env souboru a~automaticky se načítají při~spuštění aplikace pomocí management commandu. Django-allauth také poskytuje možnost propojení více OAuth2 účtů s~jedním uživatelským účtem, což zvyšuje flexibilitu pro~uživatele.
	
	Aplikace rozlišuje tři role uživatelů s~odpovídajícími oprávněními:
	\begin{itemize}
		\item \textbf{Student}~--~může prohlížet materiály, řešit testy, účastnit se diskusí a~zobrazovat vlastní statistiky.
		\item \textbf{Učitel}~--~může vše co~student + vytvářet a~spravovat obsah (předměty, okruhy, materiály, testy) a~zobrazovat statistiky všech studentů.
		\item \textbf{Administrátor}~--~má plná oprávnění včetně správy uživatelů, přístupu do~Django adminu a~Wagtail CMS.
	\end{itemize}
	
	Autorizace je kontrolována pomocí Django mixinů (LoginRequiredMixin) a~vlastních kontrol v~views. Každá view kontroluje, zda má uživatel potřebná oprávnění pro~přístup k~dané funkcionalitě.
	
	\subsection{URL routing a views}
	
	Django URL routing mapuje HTTP požadavky na~příslušné views, které zpracovávají logiku a~vracejí odpovědi. Aplikace využívá jak funkční, tak class-based views. Pro~CRUD operace jsou použity Django class-based views (CreateView, UpdateView, DeleteView, DetailView), které zjednodušují implementaci standardních operací s~daty. Vlastní logika je implementována v~metodách těchto views, například kontrola oprávnění, validace dat a~zpracování formulářů.
	
	\subsection{API endpoints}
	
	Aplikace poskytuje REST API endpoints pro~frontend, které umožňují dynamické načítání dat bez~potřeby obnovení stránky. API endpoints jsou implementovány pomocí Django views, které vracejí JSON odpovědi. 	Hlavní API endpoints zahrnují:
	\begin{itemize}
		\item Seznam předmětů a~okruhů.
		\item Statistiky studentů a~testů.
		\item Data pro~grafy (Chart.js).
		\item Informace o~materiálech a~testech.
	\end{itemize}

	\section{Frontend aplikace}
	
	Frontend aplikace je postaven na~Bootstrap~5 a~poskytuje responzivní uživatelské rozhraní, které funguje na~všech zařízeních. Frontend využívá kombinaci server-side renderingu (Django šablony) a~client-side JavaScriptu pro~dynamické načítání a~zobrazování dat.
	
	\subsection{Šablony a layout}
	
	Django šablony poskytují základní strukturu HTML stránek a~umožňují dynamické vkládání dat z~backendu. Aplikace využívá dědičnost šablon, kde základní layout je definován v~hlavní šabloně a~jednotlivé stránky ho rozšiřují. Bootstrap~5 grid systém zajišťuje responzivní layout, který se automaticky přizpůsobuje různým velikostem obrazovek. Komponenty jako navigační menu, karty, tabulky a~formuláře jsou stylovány pomocí Bootstrap CSS tříd.
	
	\subsection{Dynamické načítání obsahu}
	
	Pro~dynamické načítání a~zobrazování úloh (materiálů a~testů) je použit JavaScript modul, který komunikuje s~backend API a~načítá data bez~potřeby obnovení stránky. Tento modul umožňuje filtrování úloh podle typu, zobrazení pouze dostupných úloh a~podmíněné zobrazení tlačítek pro~správu obsahu podle role uživatele. Modul také zpracovává mazání úloh pomocí AJAX požadavků s~potvrzením uživatele.
	
	\subsection{Integrace H5P obsahu}
	
	H5P obsah je integrován do~aplikace pomocí H5P Standalone Player, který umožňuje zobrazovat H5P obsah přímo v~aplikaci bez~nutnosti externího serveru. H5P soubory (.h5p) jsou rozbalovány do~adresáře media/h5p/ a~zobrazovány pomocí JavaScript API. Standalone Player poskytuje callback funkce, které umožňují zachytit výsledky testů a~uložit je do~databáze pomocí AJAX požadavku na~backend. Toto je klíčové pro~systém statistik, protože umožňuje sledovat pokusy studentů a~vypočítávat průměrné skóre a~známky.
	
	Výhody tohoto přístupu zahrnují plnou kontrolu nad~zobrazením, možnost ukládání výsledků do~vlastní databáze, žádnou závislost na~externím serveru a~lepší výkon. Pro~starší obsah nebo obsah hostovaný na~externím serveru lze použít iframe embed kód, který je uložen v~databázi a~zobrazován přímo v~šabloně.
	
	\subsection{Vizualizace dat}
	
	Pro~zobrazení statistik výsledků testů je použita knihovna Chart.js, která poskytuje interaktivní grafy. Grafy jsou dynamicky generovány pomocí JavaScriptu na~základě dat získaných z~API. Aplikace používá sloupcové grafy pro~zobrazení výsledků testů, které zobrazují poslední výsledek a~průměrné skóre pro~každý test. Grafy jsou responzivní a~automaticky se přizpůsobují velikosti kontejneru.

	\section{Systém statistik}
	
	Systém statistik poskytuje studentům a~učitelům přehled o~výsledcích testů. Statistiky jsou vypočítávány na~základě dat uložených v~databázi při~každém pokusu studenta o~vyplnění testu. Backend poskytuje API endpoints, které vracejí agregovaná data pro~zobrazení v~frontendu.
	
	\subsection{Statistiky pro~studenty}
	
	Student vidí na~stránce předmětu přehlednou tabulku všech testů v~předmětu s~informacemi o~počtu pokusů, průměrném skóre, datu a~výsledku posledního pokusu a~automaticky vypočítané známce. Kromě tabulky je zobrazen také interaktivní graf (Chart.js), který vizualizuje výsledky testů. Graf zobrazuje poslední výsledek a~průměrné skóre pro~každý test, což umožňuje studentovi rychle identifikovat oblasti, kde potřebuje zlepšit své výsledky.
	
	\subsection{Statistiky pro~učitele}
	
	Učitel má k~dispozici přehlednou tabulku všech studentů v~předmětu s~informacemi o~průměrném skóre přes všechny testy, průměrné známce, celkovém počtu pokusů a~počtu absolvovaných testů. Učitel může kliknout na~konkrétního studenta a~zobrazit detailní statistiky, které zahrnují výsledky pro~každý jednotlivý test, včetně grafu vývoje výsledků v~čase. Toto umožňuje učiteli identifikovat studenty, kteří potřebují pomoc, a~sledovat pokrok jednotlivých studentů.
	
	\subsection{Automatické známkování}
	
	Známky jsou automaticky vypočítávány na~backendu podle procentuálního skóre z~každého pokusu. Systém používá následující škálu:
	\begin{itemize}
		\item 85--100\,\% = 1 (výborně)
		\item 70--84\,\% = 2 (chvalitebně)
		\item 55--69\,\% = 3 (dobře)
		\item 40--54\,\% = 4 (dostatečně)
		\item 0--39\,\% = 5 (nedostatečně)
	\end{itemize}
	
	Pro~každý test je vypočítána průměrná známka ze~všech pokusů studenta, což poskytuje objektivní hodnocení jeho výkonu.

	\section{Správa obsahu}
	
	Učitelé a~administrátoři mohou spravovat obsah aplikace pomocí intuitivního uživatelského rozhraní. Všechny CRUD operace jsou implementovány pomocí Django class-based views (CreateView, UpdateView, DeleteView) s~kontrolou oprávnění. Každá operace je chráněna kontrolou, zda má uživatel potřebná oprávnění pro~její provedení.
	
	\subsection{Správa předmětů a~okruhů}
	
	Správa předmětů umožňuje vytváření nových předmětů s~automatickým vytvářením slugů z~názvu, což zajišťuje čisté URL adresy. Předměty mohou být mazány s~potvrzením uživatele, přičemž systém kontroluje, zda nejsou přidruženy žádné okruhy. Okruhy jsou vytvářeny v~rámci předmětu a~mají nastavení pořadí zobrazení, což umožňuje učitelům určit pořadí, v~jakém se okruhy zobrazí studentům.
	
	\subsection{Správa testů a~materiálů}
	
	Testy jsou vytvářeny s~možností nahrání H5P souboru nebo zadání iframe embed kódu. Při~vytváření testu lze nastavit parametry jako časový limit, maximální počet pokusů a~passing score (minimální skóre pro~úspěšné absolvování). Materiály mohou být různých typů (PDF, video, H5P, odkaz, text, obrázek), přičemž každý typ má specifické možnosti zobrazení a~zpracování. Mazání testů a~materiálů je chráněno kontrolou oprávnění a~vyžaduje potvrzení uživatele.

	\section{Diskusní fórum}
	
	Každý okruh má své vlastní diskusní fórum, kde studenti a~učitelé mohou komunikovat a~sdílet informace. Fórum umožňuje vytváření nových diskusních vláken, přidávání příspěvků do~vláken a~upravování a~mazání vlastních příspěvků. Učitelé mají navíc možnost připínat důležitá vlákna a~zamykat vlákna, která již nejsou relevantní. Fórum je implementováno pomocí modelů ForumThread a~ForumPost, které jsou propojeny s~modelem Topic.

	\section{Nasazení aplikace}
	
	Aplikace je připravena pro~snadné nasazení pomocí Dockeru, což zajišťuje konzistentní prostředí pro~vývoj i~produkci. Dockerizace umožňuje zabalit aplikaci a~všechny její závislosti do~kontejneru, který lze spustit na~jakémkoli systému s~Dockerem. V~projektu jsem vytvořil Dockerfile, který definuje image aplikace včetně Python prostředí, závislostí a~konfigurace. Docker Compose pak umožňuje spustit aplikaci spolu s~PostgreSQL databází v~jednom příkazu. Výhody Dockeru zahrnují snadné nasazení, konzistentní prostředí pro~všechny vývojáře a~zjednodušenou správu závislostí.

	\chapter*{Závěr}
	\addcontentsline{toc}{chapter}{Závěr}
	
	Cílem projektu bylo vytvoření komplexní webové aplikace pro~přípravu studentů k~maturitním zkouškám z~předmětu Programové vybavení. Aplikace je postavena na~Django frameworku s~Wagtail CMS, využívá PostgreSQL databázi a~je připravena pro~nasazení pomocí Dockeru.
	
	Základem aplikace je hierarchická struktura předmětů a~okruhů, do~kterých jsou organizovány studijní materiály a~testy. Interaktivní obsah je integrován pomocí H5P Standalone Player, což umožňuje plnou kontrolu nad~zobrazením a~ukládáním výsledků.
	
	Stěžejní částí aplikace je systém statistik, který poskytuje studentům přehled o~jejich výsledcích v~testech včetně grafického zobrazení a~automatického známkování. Učitelé mají k~dispozici přehledné statistiky všech studentů s~možností zobrazit detailní informace o~jednotlivých studentech.
	
	Uživatelský systém podporuje OAuth2 autentizaci přes Google, GitHub a~Microsoft, což zjednodušuje registraci a~přihlášení. Aplikace rozlišuje tři role uživatelů (Student, Učitel, Administrátor) s~odpovídajícími oprávněními.
	
	Aplikace je zcela funkční a~připravená k~použití. Díky responzivnímu designu funguje na~všech zařízeních od~desktopových počítačů až po~mobilní telefony. Dockerizace zajišťuje snadné nasazení na~jakémkoli systému.
	
	Aplikace je zálohovaná na~GitHubu a~může být použita jako základ pro~další rozvoj vzdělávací platformy.
