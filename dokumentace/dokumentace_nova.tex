%% Abstrakt
	\noindent{\Large{\bfseries{Abstrakt}\\}}
	\noindent Výsledkem projektu je funkční webová aplikace pro~přípravu studentů k~maturitním zkouškám z~předmětu Programové vybavení. Aplikace zahrnuje registraci uživatelů přes OAuth2 (Google, GitHub, Microsoft), včetně standardních operací s~uživatelským účtem. Uživatelé si mohou prohlížet studijní materiály, řešit interaktivní testy vytvořené v~H5P a~účastnit se diskusního fóra. Stěžejní částí aplikace je systém statistik, který studentům zobrazuje přehled jejich výsledků v~testech včetně grafického zobrazení a~automatického známkování. Učitelé a~administrátoři mohou spravovat předměty, okruhy, materiály a~testy. Aplikace má zcela responzivní design a~funguje na~všech zařízeních. Aplikace je postavena na~Django frameworku s~Wagtail CMS, využívá PostgreSQL databázi a~je připravena pro~nasazení pomocí Dockeru.
	
	\vspace{18pt}
	
	\noindent{\large{\bfseries{Klíčová slova}}}
	
	\noindent webová aplikace, Django, Wagtail CMS, H5P, OAuth2, PostgreSQL, responzivní design, vzdělávací platforma
	
	\vspace{18pt}

	\noindent{\Large{\bfseries{Abstract}}}
	
	\noindent The result of this project is a functional web application for preparing students for final exams in the subject of Programming Equipment. The application includes user registration via OAuth2 (Google, GitHub, Microsoft), including standard user account operations. Users can browse study materials, solve interactive tests created in H5P, and participate in a discussion forum. The core part of the application is a statistics system that displays an overview of students' test results including graphical representation and automatic grading. Teachers and administrators can manage subjects, topics, materials, and tests. The application has a fully responsive design and works on all devices. The application is built on the Django framework with Wagtail CMS, uses PostgreSQL database, and is ready for deployment using Docker.
	
	\vspace{18pt}
	
	\noindent{\large{\bfseries{Keywords}}}
	
	\noindent web application, Django, Wagtail CMS, H5P, OAuth2, PostgreSQL, responsive design, educational platform

%% Úvod
	\chapter*{Úvod}
	\addcontentsline{toc}{chapter}{Úvod}
	
	Příprava k~maturitní zkoušce vyžaduje systematický přístup a~přehledný systém pro~organizaci studijních materiálů, testů a~sledování pokroku. Tradiční metody přípravy pomocí papírových poznámek a~tištěných materiálů jsou v~dnešní době nahrazovány moderními elektronickými řešeními, která nabízejí větší flexibilitu, interaktivitu a~možnost okamžité zpětné vazby.
	
	Mým cílem bylo vytvoření komplexní webové aplikace, která by studentům poskytla přehledný systém pro~přípravu k~maturitě z~předmětu Programové vybavení. Aplikace by měla obsahovat registraci a~přihlášení uživatelů včetně OAuth2 autentizace, umožnit správu studijních materiálů a~interaktivních testů vytvořených v~H5P, poskytovat statistické přehledy výsledků s~grafickým zobrazením a~automatickým známkováním, a~nabízet diskusní fórum pro~komunikaci mezi studenty a~učiteli.
	
	Celá webová aplikace by měla být plně responzivní, aby fungovala na~různých zařízeních od~desktopových počítačů až po~mobilní telefony. Aplikace by také měla být připravena pro~snadné nasazení pomocí Dockeru.
	
	V~této dokumentaci podrobně popisuji postup vytvoření dané aplikace. V~úvodu se zabývám návrhem databázového modelu a~architekturou aplikace. Následně řeším uživatelský systém s~OAuth2 autentizací, integraci H5P obsahu, systém statistik a~správu obsahu. V~závěru popisuji výsledky projektu a~možnosti dalšího rozvoje.

	\chapter{Teoretická a metodická východiska}

	\section{Webové vzdělávací platformy}
	
	Moderní vzdělávací platformy (Learning Management Systems~--~LMS) poskytují komplexní řešení pro~správu výukového obsahu, hodnocení studentů a~komunikaci. Mezi nejznámější patří Moodle, Canvas, Blackboard a~další. Tyto platformy obvykle zahrnují moduly pro~správu kurzů, testování, diskusní fóra a~statistiky.
	
	V~rámci tohoto projektu jsem se zaměřil na~vytvoření specializované platformy pro~přípravu k~maturitním zkouškám, která kombinuje prvky LMS s~moderními webovými technologiemi a~interaktivním obsahem.

	\section{Architektura webových aplikací}
	
	Moderní webové aplikace obvykle využívají architekturu Model-View-Controller (MVC) nebo její varianty. Django framework implementuje Model-View-Template (MVT) architekturu, kde:
	\begin{itemize}
		\item \textbf{Model}~--~reprezentuje datovou strukturu a~logiku přístupu k~datům
		\item \textbf{View}~--~zpracovává požadavky a~vrací odpovědi
		\item \textbf{Template}~--~definuje prezentaci dat uživateli
	\end{itemize}
	
	Tato architektura umožňuje oddělení logiky aplikace od~prezentace, což zajišťuje lepší udržovatelnost a~škálovatelnost kódu.

	\section{Interaktivní vzdělávací obsah}
	
	H5P (HTML5 Package) je open-source framework pro~vytváření interaktivního vzdělávacího obsahu. Umožňuje vytvářet interaktivní videa, kvízy, prezentace a~další typy obsahu, které lze snadno integrovat do~webových aplikací. H5P obsah lze zobrazit buď pomocí iframe embed kódu, nebo pomocí standalone playeru, který umožňuje větší kontrolu nad~zobrazením a~ukládáním výsledků.

	\chapter{Využité technologie}

	\section{Django Framework}
	
	Django je vysokoúrovňový Python webový framework, který podporuje rychlý vývoj a~čistý, pragmatický design. Poskytuje množství vestavěných funkcí pro~autentizaci, správu databáze, administrační rozhraní a~další. Pro~tento projekt jsem použil Django~4.2+. Hlavní výhody Django zahrnují automatickou administraci (Django Admin), ORM pro~práci s~databází, zabudovanou autentizaci a~autorizaci, šablony pro~generování HTML, URL routing a~middleware pro~zpracování požadavků.

	\section{Wagtail CMS}
	
	Wagtail je open-source CMS postavený na~Django, který poskytuje flexibilní systém pro~správu obsahu. Umožňuje vytvářet stránky s~různými typy bloků obsahu pomocí StreamField, což je ideální pro~vzdělávací obsah. Pro~tento projekt jsem použil Wagtail~5.2+.
	
	Wagtail poskytuje:
	\begin{itemize}
		\item Redakční rozhraní pro správu obsahu
		\item StreamField pro flexibilní strukturu stránek
		\item Správu obrázků a dokumentů
		\item Vyhledávání
		\item ModelViewSet API pro správu vlastních modelů
	\end{itemize}
	
	\subsection{Bližší vysvětlení využití CMS Wagtail při tvorbě webu}
	
	Wagtail CMS byl v~tomto projektu využit jako nástroj pro~správu strukturovaného obsahu a~poskytuje redakční rozhraní pro~správu stránek. Hlavní výhoda Wagtailu spočívá v~jeho flexibilním systému StreamField, který umožňuje vytvářet stránky s~různými typy bloků obsahu (text, obrázky, H5P obsah, videa atd.) bez~nutnosti programování.
	
	V~rámci projektu jsem využil Wagtail pro:
	\begin{itemize}
		\item \textbf{Správu domovské stránky}~--~Wagtail poskytuje redakční rozhraní pro~úpravu hlavní stránky aplikace, kde lze přidávat různé bloky obsahu (text, obrázky, odkazy)
		\item \textbf{Integraci s~vlastními modely}~--~Wagtail umožňuje integraci vlastních Django modelů (Subject, Topic, Material, Quiz) do~redakčního rozhraní pomocí ModelViewSet API, což umožňuje správu těchto entit přímo z~Wagtail adminu
		\item \textbf{StreamField pro~flexibilní obsah}~--~Pro~stránky okruhů lze použít StreamField, který umožňuje učitelům přidávat různé typy bloků obsahu (text, H5P, obrázky) bez~nutnosti znalosti HTML nebo programování
		\item \textbf{Správu médií}~--~Wagtail poskytuje integrovaný systém pro~správu obrázků a~dokumentů, který lze využít pro~ukládání a~zobrazování studijních materiálů
	\end{itemize}
	
	Wagtail tedy slouží jako vrstva mezi databázovými modely a~uživatelským rozhraním, která umožňuje neprogramátorům (učitelům) spravovat obsah aplikace pomocí intuitivního redakčního rozhraní, zatímco programátor definuje strukturu a~typy bloků obsahu.

	\section{PostgreSQL}
	
	PostgreSQL je pokročilý open-source relační databázový systém. Pro~produkční prostředí je PostgreSQL ideální volbou díky své spolehlivosti, výkonu a~podpoře pokročilých funkcí. Pro~vývoj jsem použil SQLite, což zjednodušuje nastavení vývojového prostředí. PostgreSQL poskytuje ACID compliance, pokročilé datové typy, full-textové vyhledávání a~JSON podporu, což je důležité pro~vzdělávací platformu s~více současnými uživateli. V~rámci projektu využívám PostgreSQL pro~ukládání všech dat aplikace pomocí Django ORM.

	\section{Bootstrap 5}
	
	Bootstrap je front-end framework pro~rychlý vývoj responzivních webových aplikací. Poskytuje předpřipravené CSS třídy a~JavaScript komponenty pro~vytváření moderních uživatelských rozhraní. Bootstrap~5 zajišťuje, že~aplikace vypadá dobře a~funguje na~všech zařízeních. Grid systém umožňuje vytvářet responzivní layouty, které se automaticky přizpůsobují různým velikostem obrazovek. V~projektu jsem využil Bootstrap~5 pro~vytvoření jednotného a~moderního vzhledu aplikace, včetně layoutu stránek, komponent karet a~responzivních tabulek.

	\section{Chart.js}
	
	Chart.js je JavaScriptová knihovna pro~vytváření grafů a~diagramů. Použil jsem ji pro~zobrazení statistik výsledků testů studentů. Chart.js poskytuje jednoduché API pro~vytváření různých typů grafů (sloupcové, čárové, koláčové atd.) s~možností interakce. Je to lehká knihovna, která nevyžaduje závislosti jako jQuery. V~projektu jsem použil sloupcové grafy pro~zobrazení výsledků testů, které jsou dynamicky generovány pomocí JavaScriptu na~základě dat získaných z~API.

	\section{H5P Standalone Player}
	
	H5P Standalone Player umožňuje zobrazovat H5P obsah bez~nutnosti externího H5P serveru. Obsahuje všechny potřebné JavaScript a~CSS soubory pro~zobrazení interaktivního obsahu. Toto řešení je ideální pro~aplikace, které chtějí mít plnou kontrolu nad~zobrazením a~ukládáním výsledků.
	
	H5P Standalone Player je distribuován jako sada JavaScript a~CSS souborů, které lze zahrnout přímo do~projektu. H5P soubory (.h5p) jsou ve~skutečnosti ZIP archivy obsahující JSON konfigurační soubory, JavaScript, CSS a~další zdroje. Standalone Player tyto soubory rozbalí a~zobrazí pomocí vlastního renderovacího engine.
	
	V~projektu jsem integroval H5P Standalone Player pro~zobrazení interaktivních testů. H5P soubory jsou uloženy v~adresáři \texttt{media/h5p/} a~jsou zobrazovány pomocí JavaScript API. Standalone Player poskytuje callback funkce, které umožňují zachytit výsledky testů a~uložit je do~databáze. Toto je klíčové pro~systém statistik, protože umožňuje sledovat pokusy studentů a~vypočítávat průměrné skóre a~známky.

	\section{Django-allauth}
	
	Django-allauth je balíček pro~Django, který poskytuje komplexní řešení pro~autentizaci, registraci a~správu účtů. Podporuje také OAuth2 autentizaci přes různé poskytovatele (Google, GitHub, Microsoft, Facebook atd.). Tento balíček výrazně zjednodušuje implementaci uživatelského systému.
	
	Django-allauth poskytuje hotové šablony a~views pro~registraci, přihlášení, odhlášení, změnu hesla, reset hesla a~ověření emailu. Pro~OAuth2 autentizaci stačí nakonfigurovat Social Applications v~Django adminu s~client ID a~secret key od~poskytovatele. Django-allauth pak automaticky zpracuje celý OAuth2 flow včetně redirectů a~získání uživatelských dat.
	
	V~projektu jsem využil django-allauth pro~implementaci autentizace přes Google, GitHub a~Microsoft. OAuth2 credentials jsou uloženy v~.env souboru a~automaticky se načítají při~spuštění aplikace pomocí management commandu. Django-allauth také poskytuje možnost propojení více OAuth2 účtů s~jedním uživatelským účtem, což zvyšuje flexibilitu pro~uživatele.

	\section{Docker}
	
	Docker je platforma pro~kontejnerizaci aplikací. Umožňuje zabalit aplikaci a~všechny její závislosti do~kontejneru, který lze spustit na~jakémkoli systému s~Dockerem. Toto řešení zajišťuje konzistentní prostředí pro~vývoj i~produkci. V~projektu jsem vytvořil Dockerfile, který definuje image aplikace včetně Python prostředí, závislostí a~konfigurace. Docker Compose pak umožňuje spustit aplikaci spolu s~PostgreSQL databází v~jednom příkazu. Výhody Dockeru zahrnují snadné nasazení, konzistentní prostředí pro~všechny vývojáře a~zjednodušenou správu závislostí.

	\chapter{Způsoby řešení a použité postupy}

	\section{Založení projektu}
	
	Projekt byl vytvořen pomocí standardního Django projektu s~následující strukturou:
	
	\begin{lstlisting}
maturitni_projekt_druhy_pokus/
├── accounts/          # Uživatelské účty a role
├── subjects/          # Předměty a maturitní okruhy
├── materials/         # Výukové materiály
├── quizzes/           # Testy a kvízy
├── forum/             # Diskusní fórum
├── maturitni_projekt/ # Hlavní konfigurace projektu
├── templates/         # HTML šablony
├── static/            # Statické soubory (CSS, JS)
└── media/             # Uživatelsky nahrané soubory
	\end{lstlisting}
	
	Každá aplikace obsahuje modely (models.py), views (views.py), URL konfiguraci (urls.py) a~šablony v~adresáři templates.

	\section{Databázový model}
	
	Databázový model aplikace je navržen tak, aby podporoval hierarchickou strukturu předmětů a~okruhů, různé typy materiálů a~testů, a~sledování pokusů studentů.
	
	Hlavní modely:
	\begin{itemize}
		\item \textbf{User}~--~rozšířený uživatelský model s~rolemi (Student, Učitel, Admin)
		\item \textbf{Subject}~--~maturitní předmět
		\item \textbf{Topic}~--~okruh v~rámci předmětu
		\item \textbf{Material}~--~výukový materiál (PDF, video, H5P, odkaz, text, obrázek)
		\item \textbf{Quiz}~--~test/kvíz s~H5P integrací
		\item \textbf{QuizAttempt}~--~pokus studenta o~vyplnění testu
		\item \textbf{ForumThread}~--~diskusní vlákno
		\item \textbf{ForumPost}~--~příspěvek ve~fóru
	\end{itemize}
	
	Vztahy mezi modely:
	\begin{itemize}
		\item Subject má více Topic (one-to-many)
		\item Topic má více Material a~Quiz (one-to-many)
		\item Quiz má více QuizAttempt (one-to-many)
		\item User má více QuizAttempt (one-to-many)
		\item ForumThread má více ForumPost (one-to-many)
	\end{itemize}

	\section{Autentizace a autorizace}
	
	\subsection{Balíček django-allauth}
	
	Pro~autentizaci jsem použil balíček django-allauth, který poskytuje:
	\begin{itemize}
		\item Registraci uživatelů
		\item Přihlášení a~odhlášení
		\item Ověření emailu
		\item Reset hesla
		\item OAuth2 autentizaci přes Google, GitHub a~Microsoft
	\end{itemize}
	
	Konfigurace OAuth2 probíhá pomocí Social Applications v~Django adminu, které jsou automaticky vytvářeny z~hodnot v~.env souboru pomocí management commandu.
	
	\subsection{Role uživatelů}
	
	Aplikace rozlišuje tři role uživatelů:
	\begin{itemize}
		\item \textbf{Student}~--~může prohlížet materiály, řešit testy, účastnit se diskusí
		\item \textbf{Učitel}~--~může vše co~student + vytvářet a~spravovat obsah (předměty, okruhy, materiály, testy)
		\item \textbf{Administrátor}~--~má plná oprávnění včetně správy uživatelů
	\end{itemize}
	
	Autorizace je kontrolována pomocí Django mixinů (LoginRequiredMixin) a~vlastních kontrol v~views.

	\section{Integrace H5P obsahu}
	
	H5P obsah je integrován do~aplikace dvěma způsoby:
	
	\subsection{H5P Standalone Player}
	
	Pro~novější implementaci jsem použil H5P Standalone Player, který umožňuje zobrazovat H5P obsah přímo v~aplikaci bez~nutnosti externího serveru. H5P soubory (.h5p) jsou rozbalovány do~adresáře media/h5p/ a~zobrazovány pomocí JavaScript API.
	
	Výhody tohoto přístupu:
	\begin{itemize}
		\item Plná kontrola nad~zobrazením
		\item Možnost ukládání výsledků do~vlastní databáze
		\item Žádná závislost na~externím serveru
		\item Lepší výkon
	\end{itemize}
	
	\subsection{Iframe embed kód}
	
	Pro~starší obsah nebo obsah hostovaný na~externím serveru lze použít iframe embed kód, který je uložen v~databázi a~zobrazován přímo v~šabloně.

	\section{Systém statistik}
	
	Systém statistik poskytuje studentům a~učitelům přehled o~výsledcích testů:
	
	\subsection{Statistiky pro~studenty}
	
	Student vidí na~stránce předmětu:
	\begin{itemize}
		\item Tabulku všech testů v~předmětu s~informacemi o:
		\begin{itemize}
			\item Počtu pokusů
			\item Průměrném skóre
			\item Datumu a~výsledku posledního pokusu
			\item Automaticky vypočítané známce
		\end{itemize}
		\item Graf výsledků testů (Chart.js) zobrazující poslední výsledek a~průměrné skóre
	\end{itemize}
	
	\subsection{Statistiky pro~učitele}
	
	Učitel vidí:
	\begin{itemize}
		\item Tabulku všech studentů v~předmětu s:
		\begin{itemize}
			\item Průměrným skóre přes všechny testy
			\item Průměrnou známkou
			\item Celkovým počtem pokusů
			\item Počtem absolvovaných testů
		\end{itemize}
		\item Možnost zobrazit detailní statistiky konkrétního studenta
	\end{itemize}
	
	\subsection{Automatické známkování}
	
	Známky jsou automaticky vypočítávány podle procentuálního skóre:
	\begin{itemize}
		\item 85--100\,\% = 1 (výborně)
		\item 70--84\,\% = 2 (chvalitebně)
		\item 55--69\,\% = 3 (dobře)
		\item 40--54\,\% = 4 (dostatečně)
		\item 0--39\,\% = 5 (nedostatečně)
	\end{itemize}

	\chapter*{Závěr}
	\addcontentsline{toc}{chapter}{Závěr}
	
	Cílem projektu bylo vytvoření komplexní webové aplikace pro~přípravu studentů k~maturitním zkouškám z~předmětu Programové vybavení. Aplikace je postavena na~Django frameworku s~Wagtail CMS, využívá PostgreSQL databázi a~je připravena pro~nasazení pomocí Dockeru.
	
	Základem aplikace je hierarchická struktura předmětů a~okruhů, do~kterých jsou organizovány studijní materiály a~testy. Interaktivní obsah je integrován pomocí H5P Standalone Player, což umožňuje plnou kontrolu nad~zobrazením a~ukládáním výsledků.
	
	Stěžejní částí aplikace je systém statistik, který poskytuje studentům přehled o~jejich výsledcích v~testech včetně grafického zobrazení a~automatického známkování. Učitelé mají k~dispozici přehledné statistiky všech studentů s~možností zobrazit detailní informace o~jednotlivých studentech.
	
	Uživatelský systém podporuje OAuth2 autentizaci přes Google, GitHub a~Microsoft, což zjednodušuje registraci a~přihlášení. Aplikace rozlišuje tři role uživatelů (Student, Učitel, Administrátor) s~odpovídajícími oprávněními.
	
	Aplikace je zcela funkční a~připravená k~použití. Díky responzivnímu designu funguje na~všech zařízeních od~desktopových počítačů až po~mobilní telefony. Dockerizace zajišťuje snadné nasazení na~jakémkoli systému.
	
	Aplikace je zálohovaná na~GitHubu a~může být použita jako základ pro~další rozvoj vzdělávací platformy.
