Internet – principy, vývoj a adresování

Principy a vývoj internetu

Internet je globální síť propojující menší sítě prostřednictvím rodiny protokolů TCP/IP.

Zrod internetu

Zrodil se koncem 60. let 20. století v USA jako vojenský projekt ARPANET, který měl zajistit decentralizovanou komunikaci

TCP/IP protokoly

V roce 1983 byla zavedena sada protokolů TCP/IP, což umožnilo standardizaci přenosu dat

Komerční využití

Na počátku 90. let byl internet zpřístupněn i komerčním subjektům, což vedlo k jeho rychlému růstu

Otcové internetu

Vint Cerf a Bob Kahn jsou považováni za "otce internetu", jelikož stáli za vznikem TCP/IP

Intranet

Obdobou Internetu je Intranet, což je uzavřená síť v rámci organizace

Adresování v internetu

IP adresa (Internet Protocol Address)

Slouží k jednoznačné identifikaci zařízení v síti

IPv4

32bitová adresa (např. 192.168.1.1), umožňuje přibližně 4,3 miliardy adres

IPv6

128bitová adresa (např. 2001:db8:85a3::8a2e:370:7334), řeší nedostatek adres v IPv4

DNS (Domain Name System)

Umožňuje překlad čitelných doménových jmen (např. google.com) na IP adresy

Domény

Domény jsou hierarchické, např.:

TLD (Top-Level Domain)

.com, .org, .cz

Second-Level Domain

google, wikipedia – domény druhé úrovně lze registrovat, v ČR např. na NIC.cz

Third-Level Domain

www, cs (cs.wikipedia.org) – subdomény třetí úrovně jsou často vytvářeny pro organizaci webu

Principy a standardy WWW, URI (URL), protokol HTTP/HTTPS – metody a stavové kódy

Principy a standardy WWW

World Wide Web (WWW) je služba internetu umožňující prohlížení hypertextových dokumentů propojených odkazy.

Tvůrce WWW

Vytvořil Tim Berners-Lee v CERNu v roce 1989

Komunikace

Základem je komunikace webového klienta (browser, prohlížeč) a webového serveru (Apache, NGINX) prostřednictvím protokolu HTTP

Bezestavový protokol

HTTP je bezestavový protokol, což znamená, že každý požadavek je nezávislý na předchozím stavu; stav je uchováván pomocí cookies nebo sessions

Komunikace HTTP

Komunikaci zahajuje klient (browser) požadavkem (request) a server odpovídá odpovědí (response).

Request (požadavek)

Obsahuje metodu (GET, POST, PUT, DELETE), hlavičky a tělo

Response (odpověď)

Obsahuje stavový kód, hlavičky a tělo odpovědi (HTML, JSON, XML)

Průběh HTTP požadavku

Krok 1

Browser pošle HTTP požadavek na server

Krok 2

Server zpracuje požadavek a odešle odpověď

Krok 3

Browser zpracuje odpověď a zobrazí obsah

Standardy WWW

HTML

HyperText Markup Language – značkovací jazyk pro tvorbu stránek

CSS

Cascading Style Sheets – jazyk pro formátování stránek

JavaScript

Skriptovací jazyk pro interaktivní webové stránky

REST

Representational State Transfer – architektura pro vytváření webových API

URI (URL) – Identifikace zdrojů

URI (Uniform Resource Identifier)

Obecné označení pro identifikaci zdrojů

URL (Uniform Resource Locator)

Specifikuje umístění zdroje na internetu (např. https://www.example.com/page#section?id=123&sort=asc)

Struktura URL

Protokol

https://

Doménové jméno

www.example.com

Cesta k souboru

/page

Fragment

#section – odkaz na konkrétní část stránky

Parametry

?id=123&sort=asc – parametry pro GET požadavek

Protokol HTTP/HTTPS

HTTP (Hypertext Transfer Protocol)

Přenosový protokol pro webové stránky; využívá port 80

HTTPS (HTTP Secure)

Zabezpečená varianta HTTP využívající TLS/SSL šifrování; využívá port 443

Šifrování – zabraňuje odposlechu dat
Důvěryhodnost – zvyšuje důvěru uživatelů
SEO – Google preferuje zabezpečené stránky

Metody HTTP

GET

Načtení dat ze serveru

POST

Odeslání dat na server

PUT

Aktualizace zdroje

DELETE

Odstranění zdroje

HEAD

Získání hlaviček odpovědi

Rozdíl mezi GET a POST

GET

Data jsou přenášena v URL, mají omezenou délku, vhodné pro načítání dat

POST

Data jsou přenášena v těle požadavku, vhodné pro odesílání dat

Stavové kódy HTTP

1xx – Informační odpovědi

100 Continue, 101 Switching Protocols

2xx – Úspěšné odpovědi

200 OK, 201 Created

3xx – Přesměrování

301 Moved Permanently, 302 Found

4xx – Chyby klienta

400 Bad Request, 404 Not Found

5xx – Chyby serveru

500 Internal Server Error, 503 Service Unavailable

Sessions, cookies a local storage

Session

Dočasné uchování uživatelských dat mezi požadavky na serveru (např. přihlášení, nákupní košík)

Cookies

Malé soubory uchovávané v prohlížeči sloužící k udržení stavu; mohou obsahovat uživatelské preference, identifikátory, atd.

HTTP cookies – ukládají se v prohlížeči a jsou odesílány s každým požadavkem na server
Session cookies – ukládají se dočasně a mazají po zavření prohlížeče
Persistent cookies – ukládají se na delší dobu

Local storage

Trvalé uložení dat v prohlížeči, přístupné pouze ze stejné domény; využívá se např. pro ukládání offline dat. Oproti cookies mají větší kapacitu a data nejsou odesílána na server
