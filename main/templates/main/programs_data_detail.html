Programové a datové soubory

Programové soubory

Programové soubory obsahují instrukce a kód potřebné k běhu aplikací.

Spustitelné soubory

Spouštějí program nebo skript

.exe, .com, .bat, .sh

Knihovny

Obsahují kód, který může být použit jinými programy

Statické knihovny – připojují se při kompilaci (např. .lib v C++)

Dynamické knihovny – načítají se za běhu programu (např. .dll v Windows, .so v Linuxu)

Skriptovací soubory

Obsahují instrukce pro interpretované jazyky

Python (.py), JavaScript (.js), Shell (.sh)

Datové soubory

Obsahují informace pro programy, nikoliv instrukce k jejich běhu.

Textové soubory

.txt, .csv, .json, .xml

Binární soubory

.dat, .bin, .db

Multimediální soubory

.jpg, .mp3, .mp4, .gif

Systémové a aplikační programy, utility a speciální software

Systémové programy

Zajišťují správu a komunikaci mezi hardwarem a softwarem. Jsou nezbytné pro běh počítače a spouštění aplikací.

Operační systémy

Základní software pro běh počítače (Windows, macOS, Linux) nebo mobilních zařízení (Android, iOS).

Real-time systémy (RTOS) – speciální typ OS pro řízení průmyslových procesů, autonomních vozidel, vesmírných sond apod. Na rozdíl od klasických OS, které mají garantovanou odezvu na události v řádu milisekund, RTOS reagují na události v řádu mikrosekund.

Ovladače zařízení

Software umožňující komunikaci mezi OS a hardwarem (např. ovladač tiskárny).

Ovladače mohou být součástí OS nebo se instalují dodatečně, např. při připojení nového zařízení (technologie Plug and Play).

Firmware

Software uložený v paměti hardware, který zajišťuje základní funkce zařízení.

Příklady: BIOS, UEFI, mikrokontroléry

Aplikační programy

Software pro konkrétní uživatelské úlohy. Mohou být určeny pro konkrétní platformu (OS) nebo běžet na více platformách (multiplatformní software).

Kancelářské balíky

Sady programů pro kancelářskou práci, obvykle obsahují textový editor, tabulkový procesor, prezentaci.

Příklady: MS Office, LibreOffice

Grafické programy

Programy pro úpravu a tvorbu grafiky a návrh modelů.

Rastrová grafika: Adobe Photoshop, GIMP

Vektorová grafika: Adobe Illustrator, Inkscape

CAD (Computer-Aided Design): AutoCAD, SolidWorks

DTP (Desktop Publishing): Adobe InDesign, Scribus

Multimediální programy

Pro přehrávání a úpravu zvuku a videa.

Příklady: VLC, Audacity, Adobe Premiere, DaVinci Resolve

Síťové aplikace

Pro komunikaci a práci s daty na internetu.

Příklady: prohlížeče, e-mailové klienty, FTP klienty

Vývojová prostředí

Programy pro vývoj software (IDE – Integrated Development Environment) pro různé programovací jazyky.

Příklady: Visual Studio Code, PyCharm

Utility

Pomocné programy pro správu a údržbu systému, zpravidla dodávané s OS nebo instalované dodatečně. Většinou jde o menší programy s jednoduchým rozhraním a omezeným účelem.

Správa souborů a disků – Průzkumník souborů, Disk Cleanup
Správa systému – Správce úloh, Správce zařízení
Antiviry – Avast, Windows Defender
Zálohovací programy – Acronis, Macrium Reflect
Komprimační programy – WinRAR, 7-Zip

Programy a autorský zákon, softwarové licence

Autorský zákon a software

Software je autorské dílo, chráněné zákonem o duševním vlastnictví. V ČR je autorské právo upraveno autorským zákonem č. 121/2000 Sb.

Autorská práva

Majitel software má právo rozhodovat o kopírování, distribuci a modifikaci programu.

Nelegální kopírování (pirátství)

Může vést k právním postihům: pokuty, zabavení majetku, vězení.

Softwarové licence

Licence je smlouva mezi vývojářem a uživatelem, která určuje podmínky používání software.

EULA

End User License Agreement

Licence pro koncového uživatele, která se zobrazuje při instalaci programu. Uživatel musí souhlasit s podmínkami, aby mohl software používat. Používá se u proprietárního software.

GNU GPL

General Public License

Licence pro open-source software, která umožňuje uživatelům volně používat, šířit a upravovat software. Uživatelé jsou povinni dodržovat podmínky licence, např. zveřejnit zdrojový kód upravené verze.

Typy softwarových licencí

Proprietární software

Placené programy, kde uživatel nemá přístup ke kódu (Windows, Photoshop).

Freemium model – základní verze zdarma a za pokročilé funkce se platí (Spotify, Evernote)

Zdrojový kód je uzavřený (closed-source), ve většině případů jde o komerční placený software

Trialware

Zkušební verze programu na omezenou dobu (obvykle na 30 dní).

Shareware

Verze programu, kterou lze vyzkoušet zdarma, ale pro plné funkce je třeba zaplatit (WinRAR, Total Commander).

Freeware

Zdarma dostupný software, který je možné používat bezplatně (Adobe Acrobat Reader, Skype).

Open-source software

Software, jehož zdrojový kód je volně dostupný a může být upravován a šířen (Linux, LibreOffice, GIMP).

Existuje mnoho typů open-source licencí, např. MIT, Apache, BSD, GNU GPL

Public domain

Software bez autorských omezení (klasické knihovny, některé starší programy).

Instalace programů, zálohování programů a dat

Instalace programů

Programy lze instalovat z instalačních médií (CD, DVD), ze sítě (internet, intranet) nebo z obchodů s aplikacemi (Google Play, App Store).

Typy instalačních balíčků

EXE

Spustitelný soubor, který obsahuje instalační program

MSI

Instalační balíček pro Windows, který obsahuje informace o instalaci (soubory, registry)

DEB, RPM

Balíčky pro Linuxové distribuce (Debian, Ubuntu, Fedora)

APP

Balíček pro macOS

Postup instalace je často automatizován pomocí instalačních průvodců (wizardů), které uživatele provedou procesem instalace.

Korektní odinstalace programu je důležitá pro odstranění všech souborů a registrů spojených s programem. Používáme k tomu utility dodávané s programem nebo vestavěné v OS, případně i speciální programy pro čištění systému (CCleaner).

Verze programů a jejich aktualizace

Verze programu

Označení konkrétního vydání programu (1.0, 2.0, 3.0). Přechod na novou verzi označujeme jako upgrade.

Aktualizace (update)

Opravy chyb, zlepšení funkcionality, nové funkce. Aktualizace mohou být minor (1.1, 1.2) nebo major (2.0, 3.0).

Aktualizace mohou být automatické (Windows Update, Google Play) nebo manuální (stahování a instalace ze stránek výrobce).

Patch

Malá aktualizace opravující konkrétní chybu nebo zabezpečení.

Service Pack

Větší aktualizace, která obsahuje mnoho oprav a nových funkcí (Windows XP SP3).

Downgrade

Vrácení na předchozí verzi programu v případě problémů s novou verzí.

Zálohování programů a dat

Proč zálohovat?

Ochrana před ztrátou dat (hardwarové selhání, ransomware, chyba uživatele).

Metody zálohování

Plná záloha

Kompletní kopie dat

Diferenciální záloha

Ukládají se pouze změny od poslední plné zálohy

Inkrementální záloha

Ukládají se pouze změny oproti předchozí záloze

Záloha systému

Záloha operačního systému a aplikací ve formě obrazu disku (disk image)

Úložiště záloh

Externí disky, NAS (Network Attached Storage) – rychlá záloha, ale zranitelná pro fyzické poškození
Zrcadlení disků (RAID) – redundantní pole nezávislých disků, které zajišťuje zálohu dat
Cloudové úložiště – záloha dat na vzdáleném serveru (Google Drive, OneDrive, Dropbox)

Software pro zálohování

Windows Backup and Restore – vestavěný nástroj v Windows
Acronis True Image – komplexní zálohovací software s možností obnovy systému
Clonezilla – alternativní open-source software pro tvorbu obrazů disků

Cloudová řešení, emulace a virtualizace

Cloud

Cloud (mračno) – označení pro vzdálené servery a služby dostupné přes internet. Umožňuje uživatelům ukládat data, spouštět aplikace a sdílet informace.

Výhody cloudu

  • Dostupnost – data jsou dostupná odkudkoliv
  • Škálovatelnost – možnost rychle zvyšovat nebo snižovat kapacitu
  • Bezpečnost – data jsou zálohována a chráněna před ztrátou
  • Spolehlivost – servery jsou monitorovány a udržovány 24/7
  • Nízké náklady – placení pouze za využitou kapacitu

Nevýhody cloudu

  • Závislost na internetu – bez připojení k internetu nelze data z cloudu získat
  • Ochrana dat – důvěra poskytovateli cloudu, riziko úniku dat
  • Výkon – rychlost přenosu dat může být omezena
  • Náklady – dlouhodobé náklady mohou být vyšší než nákup vlastního hardware

Cloudová řešení

Private cloud

Cloudové služby provozované v rámci organizace (firemní servery)

Public cloud

Cloudové služby poskytované třetí stranou (Google Cloud, Microsoft Azure)

Hybrid cloud

Kombinace privátního a veřejného cloudu

Typy modelů

SaaS

Software jako služba

Aplikace běžící v cloudu

Příklady: Google Docs, Office 365
PaaS

Platforma jako služba

Poskytuje prostředí pro vývoj aplikací, vývojáři nemusí řešit infrastrukturu

Příklady: Google App Engine, Heroku
IaaS

Infrastruktura jako služba

Pronájem výpočetních zdrojů (serverů, úložiště)

Příklady: Amazon Web Services, Microsoft Azure

Emulace

Simulace hardware nebo software na jiném systému. Umožňuje spouštět programy určené pro jiný operační systém nebo hardware.

Hardwarová emulace

Simulace fyzického hardware

Příklady: Emulátory herních konzolí - PCSX2, Dolphin

Softwarová emulace

Simulace operačního systému

Příklady: Wine pro spouštění Windows programů v Linuxu, DosBox pro staré DOS hry, BlueStacks pro spouštění Android aplikací na PC

Virtualizace

Obecně označuje vytváření virtuálních verzí fyzických zdrojů (hardware, software, úložiště).

Typy virtualizace

Hardware virtualizace

Vytváření virtuálních strojů na fyzickém hardware

Příklady: VMware, VirtualBox
Aplikační virtualizace

Izolace aplikací od operačního systému

Příklady: Docker, venv
Síťová virtualizace

Vytváření virtuálních sítí

Příklady: VPN, VLAN

Praktické využití

Vývoj software – vytvoření virtuálního prostředí pro vývoj a testování aplikací
Bezpečnost – izolace nebezpečných aplikací nebo souborů
Správa serverů – snadná migrace a zálohování serverů
Školení – vytvoření virtuálních prostředí pro školení a vzdělávání

Kvíz

Kvíz bude brzy dostupný

Zde bude interaktivní kvíz pro testování znalostí z oblasti programů a dat.
