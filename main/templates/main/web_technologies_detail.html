Webové technologie na straně klienta a serveru, frontend a backend

Model klient-server, frontend a backend

Služba WWW funguje na principu klient-server modelu, kde klient a server komunikují prostřednictvím protokolu HTTP.

Frontend (klient)

Odesílá HTTP požadavky (URL, metody GET, POST, PUT, DELETE) a zobrazuje odpovědi v prohlížeči, obvykle ve formě HTML stránek. Frontend tvoří veřejnou část aplikace, se kterou pracuje uživatel

Backend (server)

Přijímá požadavky, zpracovává data a vrací odpověď v podobě stavového kódu a obsahu (HTML, JSON, XML, obrázky, atd.). Backend zajišťuje logiku aplikace, komunikaci s databází a dalšími službami

Weboví klienti a servery

Weboví klienti

Webovými klienty mohou být prohlížeče, mobilní aplikace nebo desktopové aplikace

Webový server

Webový server může být fyzický nebo virtuální server, cloudová služba nebo serverless funkce

Základy jazyka JavaScript a tvorba klientských webových aplikací

Vývoj JavaScriptu

Vznik

Vznikl v roce 1995 jako skriptovací jazyk pro interaktivní a dynamické webové stránky

ECMAScript

Standardizován jako ECMAScript, nejnovější verze přidávají nové funkce, bývají označeny podle roku (např. ES2015 = ES6, ES2020)

Interpretace

JS je interpretovaný jazyk, běží v prohlížeči a může být také použit na serveru (Node.js); v současnosti využívá kompilaci do mezikódu (bytecode) kvůli větší rychlosti

TypeScript

Programovací jazyk rozšiřující JS o statické typování, pokročilé funkce a OOP, kompiluje se do JS, umožňuje psát robustnější kód

Vlastnosti jazyka JavaScript

Dynamické typování

Datový typ proměnné se může měnit za běhu programu, proměnné jsou typově nezávislé

Asynchronní programování

Využívá se tzv. callback funkcí, Promises a Async/Await pro efektivní zpracování asynchronních operací

Událostmi řízené programování

Reakce na uživatelské akce (kliknutí, načtení stránky) pomocí event listenerů

Základy syntaxe a datové typy v JS

Syntaxe

JS je case-sensitive, používá se camelCase syntaxe, komentáře se píšou // nebo /* ... */, nejsou povinné středníky na konci příkazů (ovšem jsou doporučeny pro čitelnost)

Proměnné

let, const, var. Rozdíl mezi nimi je v oblasti platnosti (scope), let a const jsou blokové (platí pouze v daném bloku kódu), var je funkční (platí v rámci funkce). Proměnné const nelze měnit, let a var ano

Datové typy

number, string, boolean, null, undefined, object, symbol

Příkazy

Používají se obdobně jako v jazyce C, např. if, else, for, while

Funkce

Definice pomocí function, volání pomocí názvu a závorek

Manipulace s DOM a interaktivita stránek, BOM

DOM (Document Object Model)

Reprezentace HTML stránky jako stromové struktury, umožňuje manipulaci s obsahem stránky

Selektory: výběr elementů pomocí document.querySelector(), document.getElementById(), apod.
Manipulace: změna obsahu, stylů, tříd, atributů elementů pomocí vlastností innerHTML, style, classList, setAttribute
Události: reakce na uživatelské akce (kliknutí, načtení stránky) pomocí event listenerů
Vytváření: možnost vytvářet nové elementy, mazat existující, měnit jejich vlastnosti

BOM (Browser Object Model)

Reprezentuje okno prohlížeče, umožňuje manipulaci s URL, historií prohlížení, informace o prohlížeči

Pro rozsáhlejší serverové aplikace se obvykle využívá framework Express.js, který usnadňuje vytvoření serveru, routování a zpracování požadavků

Routing

Definice cest (endpoints) a handlerů pro zpracování požadavků

Příklad routingu v Express.js

const express = require('express');
const app = express();

/* GET metoda pro získání dat */
/* Parametr req obsahuje objekt požadavku, objekt res slouží k odeslání odpovědi */
app.get('/', (req, res) => {
  res.send('Vítejte na serveru!');
});

app.listen(3000, () => console.log('Server běží na portu 3000'));

Principy synchronního vs. asynchronního programování

Synchronní kód

Blokuje běh programu a čeká na dokončení operace

Asynchronní kód

Umožňuje efektivnější zpracování požadavků, protože dokončení operace se provádí až když jsou dostupné výsledky a program může pokračovat v dalších úkolech

Analogie s restaurací

Synchronní restaurace

Číšník čeká na přípravu jídla a teprve poté přinese další objednávku

Asynchronní restaurace

Číšník přijme objednávku, předá ji kuchaři a mezitím může přijmout další objednávku

Asynchronní programování v JS

V JS se asynchronní programování využívá např. pro načítání dat ze serveru, zpracování souborů, časované úkoly.

Callback funkce

Funkce, která se zavolá po dokončení asynchronní operace

Promises

Objekt reprezentující hodnotu, která může být dostupná ihned, později nebo nikdy

Async/Await

Moderní způsob práce s asynchronním kódem, umožňuje psát asynchronní kód podobně jako synchronní

Použití Promises a Async/Await

/* Promises - zpracování asynchronní operace načtení dat */
async function fetchData() {
  /* čekání na dokončení operace funkce fetch() a získání dat z API */
  let response = await fetch('https://api.example.com/data');
  /* čekání na dokončení operace a získání dat ve formátu JSON */
  let data = await response.json();
  console.log(data);
}

Webová API, komunikace mezi frontendem a backendem, AJAX

REST API a využití ve webových aplikacích

REST (Representational State Transfer) využívá standardní HTTP metody pro výměnu dat mezi klientem a serverem.

HTTP metody

GET (načtení dat), POST (odeslání dat), PUT (aktualizace dat), DELETE (odstranění dat)

Endpoints

Zdroje jsou identifikovány pomocí URL adresy, např. /api/users, /api/products – tzv. endpoints

Stavové kódy

Informují o stavu požadavku, např. 200 (OK), 404 (Not Found), 500 (Internal Server Error)

Zásady REST API

Jednotné rozhraní, bezestavovost, možnost cachování, vrácení dat ve formátu JSON nebo XML

Využití

Ve webových aplikacích, mobilních aplikacích, IoT zařízení

Moderní technologie pro práci s API

GraphQL

Moderní dotazovací jazyk pro API, umožňuje klientovi specifikovat požadovaná data a získat je v jediném dotazu

WebSockets

Obousměrná komunikace v reálném čase mezi klientem a serverem, využívá se pro chatovací aplikace, online hry, notifikace

AJAX (Asynchronous JavaScript and XML)

Technika pro asynchronní načítání dat ze serveru bez nutnosti obnovení celé stránky (např. aktualizace obsahu zpráv v chatu, načtení nových položek v seznamu).

fetch()

Příkladem je použití funkce fetch() pro získání dat z API

Webové frameworky

Framework – soubor pravidel, nástrojů a knihoven usnadňujících vývoj aplikací, často využívají návrhové vzory a konvence.

Frontendové frameworky

React – knihovna pro tvorbu uživatelského rozhraní, využívá komponenty a virtuální DOM
Bootstrap – CSS framework pro responzivní design, využívá grid systém a komponenty, umožňuje rychlé formátování pomocí tříd. (Alternativy: Tailwind CSS, Materialize)

Backendové frameworky

Express.js – minimalistický framework pro tvorbu webových serverů v Node.js
Django – framework pro tvorbu webových aplikací v Pythonu
Flask – mikroframework v Pythonu, jednoduchý a flexibilní
Laravel – populární framework pro tvorbu webových aplikací v PHP

Full-stack frameworky

MEAN – MongoDB, Express.js, Angular, Node.js
MERN – MongoDB, Express.js, React, Node.js
Next.js – framework pro React, umožňuje server-side rendering, statické generování stránek, routování
